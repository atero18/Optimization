Pourquoi utiliser Numpy et Pandas dans ce problme ?

  • Numpy permet de faciliter les calculs "matriciels" (soustraction sur toute une ligne par exemple) et de les rendre plus rapide. Des dtails sont donns plus bas
  • Pandas permet en plus de conserver aisment le nom des noeuds et ainsi de moins s'embter avec des bijections "numro de noeud" <-> "nom de noeud"

Code principal¶

La classe principale "Little" contient diffrents algorithmes permettant de rendre le cycle hamiltonien optimal :

  • Algorithme de Little
  • Programmation Linaire avec solveur (PuLP et CBC / Gurobi)

Ici on n'utilise uniquement la matrice d'adjacence pondre du graphe. Le graphe est suppos simple et connexe, compos de n sommets. Dans beaucoup d'exemples le graphe sera galement non-orient mais les algorithmes fonctionnent de mme avec des graphes orients. Pour qu'un cycle couvrant tous les sommets existe dans un graphe, il faut que celui-ci soit fortement connexe (on peut le prouver en supposant qu'il existe un tel cycle). Il est donc possible que les algorithmes ne trouvent pas de solution, car aucun cycle, mme trs grand, n'existera.

Un travail supplmentaire intressant pourrait tre de partir d'un p-graphe et de le rendre simple en supprimant les boucles (inutiles pour notre problme) et en gardant uniquement les arrtes de taille minimale. Ce problme n'est pas difficile mais ncessite l'implmentation d'autres outils.

class Little:
    
    def __init__(self, dataF):
        self.dataF = dataF
        self.maxi = np.inf
        
    def reduc_matrice(df):
        """Rduit le df par ligne et par colonne"""
        mat = df.values
        dim = mat.shape

        somme = 0
        
        # Rduction par ligne
        for i in range(dim[0]):
            if 0 not in mat[i,:]:
                mini = np.inf
                indMin = -1
                for j in range(dim[1]):
                    if not(np.isnan(mat[i,j])) and mat[i,j] < mini:
                        indMin = j
                        mini = mat[i,j]

                if indMin != -1:
                    somme += mini
                    mat[i,:] -= mini

        # Rduction par colonne
        for j in range(dim[1]):
            if 0 not in mat[:,j]:
                mini = np.inf
                indMin = -1
                for i in range(dim[0]):
                    if not(np.isnan(mat[i,j])) and mat[i,j] < mini:
                        indMin = i
                        mini = mat[i,j]

                if indMin != -1 and mini != 0:
                    somme += mini
                    mat[:,j] -= mini

        return [df,somme]
     
        
    def calcul_regrets(df, ligne = None):
        """Calcule les regrets d'un dF"""
        regrets = np.empty(df.shape)
        regrets[:] = np.nan
        
        mat = df.values
        dim = mat.shape
        
        zeros = np.argwhere(df.values == 0)
        
        
        '''On peut ventuellement se restreindre aux regrets 
        d'une seule ligne afin d'viter des calculs inutiles'''
        if(ligne != None and ligne in df.index):
            j = 0
            noms = df.index
            while(noms[j] != ligne):
                j = j + 1
            ligne = j
        elif ligne not in range(dim[0]):
            ligne = None
        
        
        for k in zeros:
            if ligne == None or k[0] == ligne:
                regrets[k[0],k[1]] = 0

                mini = np.inf
                for i in range(dim[0]):
                    if i != k[0] and not(np.isnan(mat[i,k[1]])) \
                    and mat[i,k[1]] < mini:
                        mini = mat[i,k[1]]
                regrets[k[0],k[1]] += mini


                mini = np.inf
                for j in range(dim[1]):
                    if j != k[1] and not(np.isnan(mat[k[0],j])) \
                    and mat[k[0],j] < mini:
                        mini = mat[k[0],j]

                regrets[k[0],k[1]] += mini
        
        if ligne == None:
            return pd.DataFrame(regrets,index=df.index,
                                columns=df.columns)
        
        else:
            regrets = regrets[ligne,:]
            return regrets
    
    
    def del_ligne_col(df,i,j):
        """Supprime la ligne i et la colonne j
         de df (retourne une copie)"""
        return df.drop(index = i, columns = j)
    
    
    
    def little(self, df, liste, cout):
        "Algorithme rcursif de Little (O(e^n) !)"
        #Gestion pralable des types d'entres
        if not(isinstance(liste,list)):
            liste = [liste]
        
        # Elimination des sous-tournes
        if(len(liste) > 1):
            df.loc[liste[-1],liste[0]] = np.nan
        
        '''Rduction de la matrice et 
        obtention du cout supplmentaire'''
        [df,coutSupp] = Little.reduc_matrice(df)
        cout += coutSupp
        
        # Si le cout dpasse le max on arrte cette branche
        if(cout >= self.maxi):
            del df, liste, cout
            return None
        
        
        '''Si la matrice est de taille 2*2 cela 
        signifie qu'il ne reste que le retour'''
        if(len(df) == 2):
            self.maxi = cout
            return liste + list(set(df.index) & 
                                set(df.columns)) + [liste[0]]
        
        elif(len(df) <= 1):
            print("nbligne")
            del df, liste, cout
            return None
        
        
        '''Sinon on calcule le chemin suppos optimal
        en trouvant le regret max'''
        reg = Little.calcul_regrets(df,liste[-1])
        j = -1
        regMax = -1
        for k in range(len(reg)):
            if not(np.isnan(reg[k])) and reg[k] > regMax:
                regMax = reg[k]
                j = k
        
        
        # S'il n'y a aucun regret on arrte cette branche
        if j == -1:
            del df, liste, cout
            print("regrets")
            return None
        
        # Sinon on prend le nouveau chemin vers 'j'
        j = df.columns[j]
        df1 = Little.del_ligne_col(df,liste[-1],j)
        val1 = self.little(df1,liste + [j],cout)


        # Et on ne "prend pas" le nouveau chemin vers 'j'
        '''Si le nouveau cout (cout + regret) est plus grand 
        que le maxi on arrte la branche'''
        if cout + regMax >= self.maxi:
            val2 = None
        
        else:
            '''A noter qu'ici df2 est littralement df, 
            (mme adresse) mais cela n'a pas d'importance'''
            df2 = df
            df2.loc[liste[-1],j] = np.nan
            val2 = self.little(df2, liste,cout)
            
        
        del df, liste, cout
        
        if(val2 != None):
            return val2
        
        else:
            return val1
        
    def solve(self, depart=None):
        """Fonction de dmarrage de l'algorithme"""
        df = self.dataF.copy()
        self.maxi = np.inf
        if(depart == None or (depart not in df.columns)):
            liste = [df.columns[0]]
        else:
            liste = [depart]
        return [self.little(df, liste, 0),self.maxi]

L'algotirhme de Little cod ici est bas sur le pseudo-code propos dans le document "voyageur de commerce" joint au Projet d'Optimisation discrte ainsi que sur un cours d'introduction aux mthodes Branch & Bound de l'cole ISEN Lilles. L'algorithme est ici cod en rcursif : cela permet de rendre le programme plus rapide mais ventuellement gourmand en ressource mmoire. Pour cette raison la branche suppose "optimale" est calcule en premier afin de rduire le plus possible le nombre de branches suivantes dterminer. De plus les variables sont rgulirement supprimes afin d'conomiser de l'espace.

Il est noter que le code des fonctions en lien avec l'algorithme de Little peut sembler redondant : mise en mmoire de l'adresse des valeurs du dataFrame alors que l'on aurait pu rcuprer ces valeurs chaque calcul. Cependant il s'avre que ce code est le plus efficace que j'ai pu crire pour ce problme. En effet :

  • Si l'on passe par les valeurs sous Pandas plutt que sous Numpy, les calculs deviennent considrablement plus long (chez moi une rsolution passant par exemple de 20 50 secondes)
  • Le stockage des ensembles (range, product...) donne un code plus propre mais rend galement les calculs beaucoup plus long, jusqu' une perte de 20 secondes quand les ensembles sont grands.

La programmation linaire est ralise avec le package PuLP. Le solveur par dfaut tant CBC (solveur de PuLP), mais un autre solveur peut tre entr si l'utilisateur le souhaite.

Pour rsoudre ce problme nous utiliserons les deux matrices d'ordre n suivantes :

  • La "matrice des variables" X qui contient au plus n(n-1) variables, toutes binaires. Si l'arrte (i,j) existe, X[i][j] vaut 1 si elle est utilise dans le cycle, 0 sinon.
  • La "matrice des Poids" P contenant les poids des diffrentes arrtes. Par exemple P[i][j] contient le poids de l'arrte allant du sommet i au sommet j, si elle existe (la valeur NaN sinon).

La fonction minimiser tant donc : $$\sum_{0 \leq i\ne j \leq n-1} P[i][j]*X[i][j]$$

Cette fonction tant minimiser sous certaines conditions, ces conditions tant celles pour avoir un cycle comprenant tous les sommets. On peut traduire ces conditions par les suivantes, r-crivables sous forme linaire :

  • (1) chaque sommet servira une fois de but et une fois d'origine ;
  • (2) Il ne doit pas y avoir de sous-cycle (contrainte linaire de Miller, Tucker, Zemlin ou observation de sous-cycle a posteriori).

La condition (2) tant la plus complexe implmenter.

Pour la seconde contrainte, deux protocoles sont proposs dans le code :

  • protocole "sc" ("sous-cycle", protocole par dfaut) : les sous-cycles sont dtects a posteriori. S'il existe un sous-cycle, on ajoute une condition linaire pour le supprimer et on relance le solveur. Ce protocole, bien que souvent moins optimal que le suivant, est mis par dfaut car c'est celui que j'ai intgr naturellement. Une troisime condition est intgre ce code afin d'viter les sous-cycles "vidents" : (3) Si l'arrte (i,j) a t emprunte, l'arrte (j,i) ne pourra pas l'tre (un sommet ne peut tre but et origine pour un autre sommet).
  • protocole "MTZ" ("MillerTuckerZemlin"), qui garanti a priori la non prsence de sous-cycle. Cette contrainte est souvent plus optimale. La condition (3) indique prcdemment n'a pas besoin d'tre intgre dans ce cas.

Comme on pourra le voir dans la phase de tests, les deux protocoles ont leurs avantages en fonction des donnes du graphe.

def fileToDf(url,random=True):
    f = open(url, "r")
    dataF = pd.DataFrame()
    
    regexp = re.compile("([\w\-\s]+)(<?->)([\w\-\s]+):(\d+)")
    l = f.readline()
    while l:
        result = regexp.search(l)
        if result != None:
            dataF.loc[result.group(1),result.group(3)] = float(result.group(4))
            if(result.group(2) == "<->"):
                dataF.loc[result.group(3),result.group(1)] = float(result.group(4))
        l = f.readline()
    if random:
        order = list(range(len(dataF)))
        shuffle(order)
        dataF = dataF.iloc[order,:]
    dataF = dataF.loc[:,dataF.index]
    return dataF

def creation_graphe_alea(n=40,oriente=False, dMax = 100):
    seed(time())
    mat = np.zeros((n,n))
    mat += np.nan
    for i,j in product(range(0,n),range(0,n)):
        if i != j and np.isnan(mat[i][j]):
            mat[i][j] = 1 + int(random()*dMax)
        if not(oriente):
            mat[j][i] = mat[i][j]
            
    liste = []
    for i in range(n):
        liste += [str(i)]
    return pd.DataFrame(mat, index=liste, columns=liste)

    '''

                        gurobiLect["scNB"][i] += 1
                    elif nomData == "GUROBI-MTZ":
                        gurobiLect["MTZ"][i] += numb
                        gurobiLect["MTZNB"][i] += 1

def test_temporel_PL(nomFichier,n,nbBoucles,doCbc=['sc','MTZ'], doGurobi=['sc','MTZ'],affichage='liste'):
    '''
    doCBC : None si aucun calcul ne doit tre fait sous CBC
            sinon mettre True pour le type de protocole  utiliser
    doGurobi : None si aucun calcul ne doit tre fait sous Gurobi
            sinon mettre True pour le type de protocole  utiliser
    '''
    
    
    for k in range(nbBoucles):
        infoDepart = "tour " + str(k+1) +  "/" +str(nbBoucles) + " :"
        dataF = creation_graphe_alea(max(n))
        
        clear_output(True)
        
        if doCbc is not None:
            cbcCalcul = {'sc' : [0]*len(n), 'MTZ' : [0]*len(n)}
        else:
            cbcCalcul = None
        
        if doGurobi is not None:
            gurobiCalcul = {'sc' : [0]*len(n), 'MTZ' : [0]*len(n)}
        else:
            gurobiCalcul = None
        
        print(infoDepart,end="")
        
        for i in range(len(n)):
            size = n[i]
            if affichage == "liste":
                print(size, end=" ")
                
            elif affichage == "ratio":
                clear_output(True)
                print(infoDepart, str(i+1), "/", str(len(n)), sep=" ",end="")
                
            d = dataF.iloc[:size,:size]

            # Partie CBC
            if doCbc is not None:
                if 'sc' in doCbc:
                    p = PL_voyage(d)
                    s = time()
                    res = p.solve(protocole="sc",solv=None)
                    s = round(time() - s,4)
                    if isinstance(res,str):
                        print("UNS-CBC (sc)",end = " ")
                    else:
                        cbcCalcul['sc'][i] += s
                
                if 'MTZ' in doCbc:
                    p = PL_voyage(d)
                    s = time()
                    res = p.solve(protocole="MTZ",solv=None)
                    s = round(time() - s,4)
                    if isinstance(res,str):
                        print("UNS-CBC (MTZ)",end = " ")

Le globe trotter¶

Cet exemple a t imagin par moi-mme afin de prendre en compte un graphe orient. Le modle propos est plus que discutable et sert uniquement titre d'exemple / cas d'cole.

Un globe-trotter franais souhaite visiter 4 capitales diffrentes en partant de Paris :

  • Reykjavik (Islande)
  • Kuala Lumpur (Malaisie)
  • Sri Jayawardenapura Kotte (Sri Lanka)
  • Islamabad (Pakistan)

Il a tout le temps qu'il souhaite pour faire son voyage. Cependant il souhaite minimiser l'argent qu'il mettra dans le transport. On suppose que le prix d'un billet de transport pour aller d'une capitale une autre dpend :

  • de la distance entre les deux capitales
  • du PIB / habitant du pays de dpart (d'o l'aspect orient)

Le prix (en ) d'un billet est donc modlis par cette relation :

$$prix = C * d * \frac{P_{dpart}}{P_{min}}$$

Avec :

  • C le facteur de rduction ;
  • d la distance entre les deux capitales ;
  • $P_{dpart}$ le PIB / habitant du pays de dpart ;
  • $P_{min}$ le plus petit PIB / habitant dans le monde.

L'ensemble des donnes utilises pour cet exemple ainsi que les sources se trouvent dans le fichier "globe trotter.xlsx". La matrice d'adjacence de ce problme est la suivante (les prix tant en ) :
